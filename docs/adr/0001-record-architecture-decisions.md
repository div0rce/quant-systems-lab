# ADR 0001: Record Architecture Decisions

## Status

Accepted

## Context

We need to record architectural decisions made in this project so that future readers (including recruiters and interviewers) can understand the reasoning.

## Decision

We will use Architecture Decision Records (ADRs) in `docs/adr/`.

Each ADR documents: status, context, decision, and consequences.

## Consequences

- Decisions are traceable
- Reviewers can understand why choices were made
- The project demonstrates disciplined engineering process
